## 1.2 함수

### 함수란 무엇인가?

함수는 자바스크립트에서 가장 중요한 개념 중 하나이다.
함수는 재사용 가능한 코드 블록으로, 특정 작업을 수행하는 목적으로 설계된 코드의 집합이다.
함수는 입력을 받아 출력을 반환하는 역할을 한다.

리액트에서의 함수는 단순히 값을 계산하는 역할 뿐만 아니라 JSX를 반환하는 역할도 한다.

### 함수 선언의 4가지 방법

자바스크립트에서 함수를 선언하는 방법은 크게 4가지로 구분할 수 있다.

- 함수 선언문

```ts
function add(a, b) {
  return a + b;
}
```

위와 같은 방법으로 함수를 표현식이 아닌 문으로 표현한 방식이다.
흔히 무언가를 산출해내는 구문을 표현식 이라고 구분하나, 함수 선언문은 표현식이 아닌 문이다.

- 함수 표현식

```ts
const add = function (a, b) {
  return a + b;
};
```

이게 바로 표현식이다. 함수 선언을 통해 add 라는 변수가 표현되었다.

그렇다면 이 두가지 방법은 어떤 차이점을 가지고 있는가?
가장 큰 차이점은 호이스팅이다.
호이스팅의 실체는 조금 더 깊게 다뤄봐야 할 개념이지만, 단순히 설명할 때 호이스팅은 변수의 선언이 스코프의 최상단으로 끌어올려지는 현상을 의미한다.
이 현상은 함수 선언문과 함수 표현식에서 다르게 동작한다.
함수 선언문은 호이스팅이 발생하면 스코프의 최상단으로 끌어올려지는 반면, 함수 표현식은 호이스팅이 발생하더라도 함수 표현식 자체가 끌어올려지지 않는다.

- Function 생성자 함수

```ts
const add = new Function("a", "b", "return a + b");
```

이 방법은 많은 방면으로 권장되지 않는 방법이다. 일반적으로 사용되지 않아 가독성을 해칠 뿐만 아니라, 클로저또한 생성되지 않기 때문에 성능상의 이점도 없다.

- 화살표 함수

```ts
const add = (a, b) => a + b;
```

화살표 함수의 가장 큰 장점은 간결한 문법이다.
화살표 함수만의 특징은 constructor를 사용할 수 없다는 점과 arguments 객체를 사용할 수 없다는 점 그리고 this 바인딩 방식이 다르다는 점이다.

constructor가 없으므로 생성자 함수로 사용할 수 없다.
arguments 객체는 함수 내부에서 사용되는 객체로, 함수에 전달된 인수들을 배열 형태로 저장하는 객체이다. 화살표 함수는 일반적인 함수와 달리 arguments 객체를 가지고 있지 않다.
this 바인딩 방식은 화살표 함수 내부에서 this가 가리키는 객체가 함수가 선언된 위치에 따라 결정된다는 점이다.

### 다양한 함수들

- 고차함수
  고차함수는 함수를 인자로 받거나 함수를 반환하는 함수를 말한다.

- 즉시실행함수 IIFE
  IIFE는 함수 선언과 동시에 즉시 실행되는 함수를 말한다.

### 함수 생성에서 주의할 점

클린코드의 그것과 유사한데, 부수효과를 억제하고 함수를 작게 만들며 이름을 잘 지어야 한다.
